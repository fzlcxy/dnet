# 变更：添加协议配置GUI工具

## 为什么需要这个变更
目前，开发者无法清楚地知道当发送某个C2S协议后，服务器会回复哪些S2C协议。.dnet协议文件中C2S和S2C的对应关系是隐式的，这给开发和测试带来了困扰。需要一个可视化工具来配置和管理这些映射关系。

## 变更内容
- 基于Tkinter开发GUI配置工具，用于配置C2S到S2C的响应映射关系
- 在`clientconfig/`目录下创建JSON配置文件，目录结构与`proto/`目录保持一致
- GUI脚本存放在`clientscript/`目录
- 支持以下功能：
  - 浏览和加载.dnet文件
  - 编辑C2S回包配置（一个C2S可以对应多个S2C）
  - 配置验证（检查配置的S2C是否存在于对应的.dnet文件中）
  - 批量导入/导出配置
- 一个.dnet文件的所有C2S配置存储在一个对应的JSON配置文件中

## 影响范围
- 影响的规范：protocol-config-gui（新增能力）
- 影响的代码：
  - 新增目录：`clientconfig/`（存储JSON配置文件）
  - 新增目录：`clientscript/`（存储GUI Python脚本）
  - 使用现有：`proto/`目录（.dnet协议文件保持不变）
